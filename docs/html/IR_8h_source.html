107 /* A few important comments.
108  IRInstr has no jump instructions.
109  cmp_* instructions behaves as an arithmetic two-operand instruction (add or mult),
110  returning a boolean value (as an int)
111 
112  Assembly jumps are generated as follows:
113  BasicBlock::gen_asm() first calls IRInstr::gen_asm() on all its instructions, and then
114  if exit_true is a nullptr,
115  the epilogue is generated
116  else if exit_false is a nullptr,
117  an unconditional jmp to the exit_true branch is generated
118  else (we have two successors, hence a branch)
119  an instruction comparing the value of test_var_name to true is generated,
120  followed by a conditional branch to the exit_false branch,
121  followed by an unconditional branch to the exit_true branch
122  The attribute test_var_name itself is defined when converting
123  the if, while, etc of the AST to IR.
124 
125 Possible optimization:
126  a cmp_* comparison instructions, if it is the last instruction of its block,
127  generates an actual assembly comparison
128  followed by a conditional jump to the exit_false branch
129 */
